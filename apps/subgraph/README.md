# Subgraph

The Graph subgraph for indexing OctantToken (ERC-20) events. Runs a local Graph Node via Docker and deploys the subgraph automatically.

## Scripts

| Command | Description |
|---------|-------------|
| `pnpm dev` | Start Graph Node, build, and deploy the subgraph |
| `pnpm build` | Codegen + build the subgraph (no deploy) |
| `pnpm codegen` | Generate AssemblyScript types from schema |
| `pnpm clean` | Remove build and generated artifacts |

## Development (local)

The subgraph is typically started as part of the full dev stack:

```bash
# From monorepo root — starts chain + subgraph + web
pnpm dev:web
```

Or run it standalone (requires local anvil on port 8545):

```bash
# Terminal 1: start anvil + deploy contracts
pnpm chain:dev

# Terminal 2: start Graph Node + deploy subgraph
pnpm subgraph:dev
```

### What `dev.sh` does

1. Checks Docker is running
2. Waits for anvil on port 8545
3. Starts Graph Node infrastructure (graph-node, IPFS, Postgres) via docker-compose
4. Waits for Graph Node to be ready
5. Runs `graph codegen` and `graph build --network localhost`
6. Creates and deploys the subgraph to the local Graph Node
7. Keeps alive until Ctrl+C (cleanup stops Docker containers)

### Verify

```bash
# Check Graph Node is serving queries
curl -s http://localhost:8000/subgraphs/name/octant-token \
  -H 'Content-Type: application/json' \
  -d '{"query":"{ transfers(first: 1) { id } }"}' | jq .
```

## Staging (Tenderly fork)

Staging indexes from a Tenderly mainnet fork instead of local anvil. From the monorepo root:

```bash
pnpm staging
```

This reads `.env.staging` which sets `GRAPH_ETHEREUM_RPC` to point Graph Node at the Tenderly fork RPC. No local anvil is started.

### How it works

1. `GRAPH_ETHEREUM_RPC` env var overrides the Graph Node `ethereum` config in `docker-compose.yml`
2. `dev.sh` detects `GRAPH_ETHEREUM_RPC` is set and skips the anvil wait step
3. Graph Node connects directly to the Tenderly fork RPC for indexing
4. The `localhost` network name is preserved — subgraph.yaml and graph build commands stay unchanged

### Setup

1. **Create `.env.staging`** at the monorepo root with your Tenderly RPC URL:

   ```bash
   GRAPH_ETHEREUM_RPC=localhost:https://mainnet.gateway.tenderly.co/YOUR_KEY
   VITE_MAINNET_RPC_URL=https://mainnet.gateway.tenderly.co/YOUR_KEY
   VITE_SUBGRAPH_URL=http://localhost:8000/subgraphs/name/octant-token
   ```

   Note: `GRAPH_ETHEREUM_RPC` uses the `localhost:` prefix (the Graph Node network name) even though it points to Tenderly.

2. **Write `networks.json`** with the contract address and start block from your Tenderly fork:

   ```json
   {
     "localhost": {
       "OctantToken": {
         "address": "0xYourDeployedContractAddress",
         "startBlock": 12345678
       }
     }
   }
   ```

3. **Run staging**:

   ```bash
   pnpm staging
   ```

## Docker Services

The `docker-compose.yml` runs three containers:

| Service | Image | Ports | Purpose |
|---------|-------|-------|---------|
| graph-node | graphprotocol/graph-node | 8000, 8001, 8020, 8030 | GraphQL API + indexing |
| ipfs | ipfs/kubo:v0.27.0 | 5001 | Subgraph manifest storage |
| graph-postgres | postgres:16 | 5433 | Graph Node state storage |

### Manual Docker management

```bash
# Start infrastructure without deploying subgraph
pnpm subgraph:up

# Stop infrastructure
pnpm subgraph:down

# View Graph Node logs
docker compose -f apps/subgraph/docker-compose.yml logs graph-node -f
```

## Project Structure

```
apps/subgraph/
  schema.graphql          # GraphQL schema (Transfer, Approval entities)
  subgraph.yaml           # Subgraph manifest (data sources, handlers)
  src/mapping.ts          # Event handler mappings
  abis/                   # Contract ABIs
  networks.json           # Contract addresses per network (generated by chain dev)
  docker-compose.yml      # Graph Node infrastructure
  scripts/dev.sh          # Dev/staging startup script
```

## Environment Variables

| Variable | Where | Purpose | Default |
|----------|-------|---------|---------|
| `GRAPH_ETHEREUM_RPC` | `docker-compose.yml` | Override Graph Node's Ethereum RPC endpoint | `localhost:http://host.docker.internal:8545` |

Set via `.env.staging` at the monorepo root for staging mode.
