# @octant/validation

Zod validation schemas generated from Prisma schema. **Zero Prisma runtime dependency** - safe for frontend use.

## Installation

```bash
pnpm add @octant/validation
```

## Usage

### Pure Model Schemas

For validating complete model objects (e.g., data from API responses):

```typescript
import { UserModelSchema, ProductModelSchema } from '@octant/validation';

// Validate API response
const user = UserModelSchema.parse(apiResponse);

// Type-safe access
console.log(user.email, user.name);
```

### Input Schemas

For validating raw input data (includes all fields):

```typescript
import { ProductInputSchema } from '@octant/validation';

// Full schema with all fields
const product = ProductInputSchema.parse(data);
```

### Business Validation Schemas

For create/update operations with business rules:

```typescript
import { ProductValidationSchema } from '@octant/validation';

// Includes business rules:
// - name: required, non-empty
// - description: required, non-empty
// - priceInCents: must be non-negative
const validProduct = ProductValidationSchema.parse({
  name: 'Widget',
  description: 'A useful widget',
  priceInCents: 1999,
  category: 'ELECTRONICS',
});

// For partial updates
const partialUpdate = ProductValidationSchema.partial().parse({
  priceInCents: 2499,
});
```

### Type Exports

```typescript
import type {
  UserPureType,
  ProductPureType,
  ProductValidationInput,
  ProductCategory
} from '@octant/validation';

// Use in React state
const [user, setUser] = useState<UserPureType | null>(null);
```

### Enum Schemas

```typescript
import { ProductCategorySchema } from '@octant/validation';

// Validate category
const category = ProductCategorySchema.parse('ELECTRONICS');

// Get all valid values
const categories = ProductCategorySchema.options;
// ['ELECTRONICS', 'CLOTHING', 'FOOD', 'OTHER']
```

## Architecture

```
@octant/validation
├── src/
│   ├── index.ts              # Re-exports (manually maintained)
│   └── generated/            # Auto-generated by prisma-zod-generator
│       └── schemas/
│           ├── variants/
│           │   ├── pure/     # Model schemas (Prisma-free)
│           │   └── input/    # Input schemas (Prisma-free)
│           └── enums/        # Enum schemas (Prisma-free)
└── dist/                     # Compiled output
```

### Why This Structure?

The `prisma-zod-generator` produces multiple schema types:

1. **Variants (pure/input)** - Clean Zod schemas, no Prisma dependency
2. **CRUD Operations** - Type-safe against `Prisma.*` types, requires `@prisma/client`
3. **Enums** - Clean Zod enums, no Prisma dependency

We **only export** variants and enums to keep this package Prisma-free for frontend use.

## Regenerating Schemas

Schemas are auto-generated from `packages/db/prisma/schema.prisma`:

```bash
# From monorepo root
pnpm --filter @octant/db db:push     # Push schema + regenerate
pnpm --filter @octant/db db:generate # Just regenerate (no schema push)
```

After regeneration, the `src/generated/` directory is updated automatically.

## Adding Business Validation Rules

The generated schemas provide type validation only. For business rules, extend them in `src/index.ts`:

```typescript
import { z } from 'zod';
import { ProductInputSchema as GeneratedSchema } from './generated/schemas/variants/input/index';

export const ProductValidationSchema = GeneratedSchema
  .omit({ id: true, createdAt: true, updatedAt: true })
  .extend({
    name: z.string().min(1, 'Name is required'),
    priceInCents: z.number().int().nonnegative('Price must be non-negative'),
  });
```

## Version Compatibility

> **CRITICAL**: This package depends on generated output from `prisma-zod-generator`. Version compatibility is strict.

| Package | Version | Notes |
|---------|---------|-------|
| `zod` | `^3.24.0` | Runtime dependency |
| `prisma` | `6.19.2` | **PINNED** - v7.x does not support MongoDB |
| `prisma-zod-generator` | `1.31.8` | **PINNED** - v2.x requires Prisma 7 |
| `@prisma/client` | `6.19.2` | In `@octant/db`, not here |

### Why Pinned Versions?

1. **Prisma 7.x** - Does not support MongoDB yet (as of Jan 2026)
2. **prisma-zod-generator 2.x** - Requires Prisma 7, incompatible with our MongoDB setup
3. **prisma-zod-generator 1.32.x** - Has bug with `@prisma/internals@7` dependency

See [ADR-002](../../docs/adr/ADR-002-prisma-mongodb-setup.md) for full details.

## TypeScript Configuration

This package uses `moduleResolution: "Bundler"` because `prisma-zod-generator` outputs imports without `.js` extensions:

## Troubleshooting

### "Cannot find module '@prisma/client'"

The generated CRUD operation schemas import `@prisma/client`. This package only exports the Prisma-free schemas. If you see this error:

1. Check that `tsconfig.json` `include` only covers `variants/` and `enums/`
2. Ensure you're importing from the package entry point, not directly from `generated/`

### Schemas out of sync with Prisma schema

Run `pnpm --filter @octant/db db:push` to regenerate after schema changes.

### Validation not catching invalid data

Generated schemas only validate types. For business rules (min length, non-negative, etc.), use the extended schemas like `ProductValidationSchema` or create your own with `.extend()`.
